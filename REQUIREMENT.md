# pile — 정제 요구사항 정의서

> 소규모 그룹(스터디, 팀, 모임)이 로그인 없이 자료를 빠르게 쌓고 공유하는 실시간 협업 보드 앱.

이 문서는 `REQUIREMENT.md`를 최우선 기준으로 하고, `PRD.md`와 `mockup/`의 화면·동작 세부사항을 합쳐 정제한 구현 요구사항이다. 문서 간 충돌이 있을 때는 `REQUIREMENT.md`를 우선한다. `PRD.md`에만 있는 현재 동작 규칙은 `REQUIREMENT.md`와 충돌하지 않는 범위에서 보강 요구사항으로 반영한다.

---

## 0. 기준과 해석 원칙

### 0-1. 우선순위

1. `REQUIREMENT.md`
2. `mockup/`의 동작, 화면, 샘플 데이터, 스타일
3. `PRD.md`

### 0-2. 주요 정제 결정

* 기본 채널은 `REQUIREMENT.md` 기준으로 **일반 / 과제 / Q&A / 자료실** 4개다.
* 보드가 아직 없을 때 첫 아이템 생성으로 보드가 자동 생성되는 규칙은 `PRD.md`에서 보강한다.
* `PRD.md`의 단순 보드 요구사항보다 `REQUIREMENT.md`와 mockup의 실시간 협업, 투표, 고정, 리액션, Tweaks 요구사항을 우선한다.
* mockup에서 확인되는 디자인/상호작용 세부사항은 실제 제품 요구사항으로 본다.
* 자동 테스트나 내부 구현 파일 구조는 이 문서의 범위가 아니다. 이 문서는 제품 동작과 사용자 경험을 정의한다.

## 1. 제품 목표와 범위

### 1-1. 목표

pile은 스터디, 팀 활동, 모임처럼 작은 그룹이 자료를 빠르게 모으고 함께 보는 협업 보드다. 사용자는 계정을 만들지 않고도 보드 URL에 접속해 텍스트, 링크, 이미지, 파일, 투표를 올릴 수 있어야 한다.

### 1-2. 핵심 경험

* 사용자는 보드 URL을 공유받아 바로 입장한다.
* 사용자는 페이지 어디서나 붙여넣기 또는 드래그 앤 드롭으로 자료를 올릴 수 있다.
* 올린 자료는 채널별 피드에 카드 형태로 쌓인다.
* 여러 접속자는 같은 보드에서 새 아이템, 이름 변경, 커서 위치를 실시간으로 공유한다.
* 관리자는 운영자 역할로 아이템 고정, 삭제, 채널 추가를 수행한다.
* 보드는 “더미(pile)” 은유를 시각적으로 드러내며, 카드가 쌓이고 고정 카드가 겹쳐 보이는 느낌을 제공한다.

### 1-3. 비범위

* 일반 사용자 계정 가입/로그인
* 역할 세분화된 권한 체계
* WYSIWYG 편집기
* 모바일 네이티브 앱
* 복잡한 문서 공동 편집
* 장기 보존용 지식관리 시스템

## 2. 보드와 URL

### 2-1. 보드 식별

* 보드는 고유 board id 또는 slug로 식별된다.
* 사용자에게 표시되는 대표 URL 형식은 `pile.so/<board-id>`다.
* mockup의 기본 보드 예시는 `pile.so/frontend-101`이다.
* 앱 라우트에서는 `/:boardId` 형식으로 보드에 접근한다.

### 2-2. 보드 URL 표시와 복사

* 상단바 왼쪽에 서비스 로고와 보드 URL 버튼을 표시한다.
* 보드 URL 버튼은 host 영역과 board id 영역을 구분해 보여준다.
  * 예: `pile.so/` + `frontend-101`
* 보드 URL 버튼을 클릭하면 보드 URL을 클립보드에 복사한다.
* 복사 성공 시 하단 중앙 토스트를 표시한다.
  * 문구: `보드 URL을 복사했어요`
* URL 버튼에는 복사 아이콘을 표시한다.

### 2-3. 보드 자동 생성

* 존재하지 않는 보드 URL에 접근해도 보드 화면을 보여줄 수 있다.
* 해당 화면에서 사용자가 첫 아이템을 생성하면 보드가 자동 생성된다.
* 자동 생성 시 기본 채널 세트를 함께 준비한다.
* 자동 생성 보드의 표시 이름은 board id를 기본값으로 사용한다.

## 3. 채널

### 3-1. 기본 채널

* 새 보드는 다음 기본 채널을 가진다.
  * `일반`
  * `과제`
  * `Q&A`
  * `자료실`
* 기본 선택 채널은 `일반`이다.

### 3-2. 채널 탭

* 채널은 보드 상단, 상단바 아래에 chip 형태의 탭으로 표시한다.
* 채널 탭은 가로 스크롤 가능해야 한다.
* 스크롤바는 시각적으로 숨길 수 있다.
* 각 채널 탭에는 해시(`#`) 아이콘을 표시한다.
* 현재 선택된 채널은 배경, 테두리, 그림자, 굵기 등으로 활성 상태를 표시한다.
* 채널 탭에는 해당 채널의 아이템 수를 배지로 표시한다.
* 아이템 수가 0이면 배지를 숨길 수 있다.
* 채널을 클릭하면 해당 채널의 피드만 표시한다.

### 3-3. 채널 추가

* 관리자는 채널을 새로 추가할 수 있다.
* 채널 추가 버튼은 채널 탭 목록의 끝에 표시한다.
* 버튼 라벨은 `채널` 또는 `+ 채널` 계열로 표시한다.
* 채널 추가 버튼 클릭 시 인라인 입력 상태로 전환한다.
* 입력 placeholder는 `채널 이름`이다.
* Enter를 누르면 입력값으로 채널을 생성한다.
* Escape를 누르면 입력을 취소한다.
* 빈 이름은 생성하지 않는다.
* 채널 생성 직후 새 채널로 자동 전환한다.
* 채널 생성 성공 시 토스트를 표시한다.
  * 문구 형식: `#<채널명> 채널을 만들었어요`

## 4. 사용자와 권한

### 4-1. 일반 사용자

* 사용자는 계정 없이 익명으로 참여한다.
* 최초 참여 시 익명 닉네임을 가진다.
* 기본 닉네임은 동물 기반 이름을 사용한다.
  * mockup 예시: `느긋한 펭귄`, `조용한 다람쥐`, `성실한 두루미`, `호기심 여우`
* 상단바에 현재 사용자 chip을 표시한다.
* 사용자 chip을 클릭하면 표시 이름 인라인 편집 상태로 전환한다.
* 이름 입력 중 Enter를 누르면 확정한다.
* 이름 입력 중 Escape를 누르면 취소한다.
* 사용자는 본인이 올린 아이템만 삭제할 수 있다.
* 본인이 올린 아이템 카드에는 `나` 배지를 표시한다.

### 4-2. 관리자 모드

* 상단바 오른쪽에 `관리자 로그인` 버튼을 표시한다.
* 관리자 모드가 활성화되면 버튼이 강조 상태로 바뀐다.
* 관리자 버튼에는 방패 아이콘을 표시한다.
* 관리자 모드 활성화 시 토스트를 표시한다.
  * 켬: `관리자 모드 · 모든 아이템 관리 가능`
  * 끔: `관리자 모드를 껐어요`
* 관리자는 다음 권한을 가진다.
  * 모든 아이템 삭제
  * 아이템 고정
  * 아이템 고정 해제
  * 새 채널 추가
* 관리자 사용자의 아바타에는 방패 아이콘을 표시한다.
* 관리자 작성 카드에는 `관리자` 배지를 표시한다.

### 4-3. 아바타

* 일반 사용자 아바타는 이름 첫 글자를 표시한다.
* 아바타 배경색은 사용자명 기반으로 결정론적으로 생성한다.
* 일반 사용자 아바타 색상은 warm 계열 색조를 사용한다.
* 관리자는 일반 이니셜 대신 방패 아이콘을 사용한다.
* 실시간 접속자 표시용 peer pip은 각 peer별 결정론적 색상을 사용한다.

## 5. 컴포저와 입력

### 5-1. 컴포저 위치

* 컴포저는 피드 상단에 sticky로 고정한다.
* 피드를 스크롤해도 컴포저는 상단에 유지된다.
* 컴포저 배경은 피드 배경과 자연스럽게 이어지도록 그라데이션 처리할 수 있다.

### 5-2. 접힌 상태

* 기본 상태는 접힌 상태다.
* 접힌 상태에서는 큰 붙여넣기 힌트 버튼으로 표시한다.
* 버튼에는 키보드 힌트 `⌘ V` 또는 이에 준하는 붙여넣기 힌트를 표시한다.
* 버튼 문구는 다음 의미를 포함한다.
  * `여기에 붙여넣기`
  * `텍스트 · 링크 · 파일을 던져 두세요`
* 오른쪽에는 붙여넣기 CTA를 표시한다.
* 접힌 컴포저 클릭 시 펼친 상태로 전환한다.

### 5-3. 펼친 상태

* 펼친 상태는 텍스트에리어와 하단 액션 영역으로 구성한다.
* 텍스트에리어는 자동 포커스된다.
* placeholder는 텍스트/링크 붙여넣기와 Markdown 지원을 안내한다.
* 내용이 비어 있으면 제출 버튼을 비활성화한다.
* `Cmd+Enter` 또는 `Ctrl+Enter`로 제출한다.
* Escape를 누르면 작성 중인 내용을 버리고 접힌 상태로 돌아간다.
* 취소 버튼을 누르면 작성 중인 내용을 버리고 접힌 상태로 돌아간다.
* 제출 버튼은 `올리기` 라벨과 단축키 힌트를 포함한다.
* 하단 안내 문구는 자동 분류 규칙을 설명한다.
  * 텍스트는 텍스트 아이템
  * URL은 링크 아이템
  * 이미지는 파일 아이템

### 5-4. 자동 분류

* 제출 내용이 URL 하나만 포함하면 링크 아이템으로 분류한다.
* 그 외 텍스트는 텍스트 아이템으로 분류한다.
* 클립보드 이미지 또는 파일 드롭은 파일 아이템으로 분류한다.
* URL 판별은 공백 없는 `http://` 또는 `https://` URL을 기준으로 한다.

### 5-5. 전역 붙여넣기

* 페이지 어디서나 붙여넣기를 지원한다.
* 텍스트에리어 또는 input에 포커스가 있을 때는 전역 붙여넣기를 가로채지 않는다.
* 클립보드 이미지가 있으면 파일 아이템으로 즉시 추가한다.
* 클립보드 텍스트가 있으면 자동 분류 규칙에 따라 텍스트 또는 링크 아이템으로 즉시 추가한다.
* 이미지 붙여넣기 성공 시 토스트를 표시한다.
  * 문구: `이미지를 붙여넣었어요`

## 6. 드래그 앤 드롭

### 6-1. 파일 드래그

* 파일을 보드 위로 드래그하면 전체 화면 드롭 오버레이를 표시한다.
* 오버레이는 흐린 배경, 점선 테두리, 중앙 안내 카드로 구성한다.
* 중앙 안내 카드에는 다운로드/드롭 아이콘, 제목, 설명을 표시한다.
* 제목 예시: `여기에 놓으세요`
* 설명은 파일은 미리보기 또는 다운로드로, 텍스트는 아이템으로 쌓인다는 의미를 전달한다.

### 6-2. 드롭 처리

* 파일을 드롭하면 파일 아이템으로 추가한다.
* 여러 파일을 드롭하면 각 파일을 개별 아이템으로 추가한다.
* 여러 파일은 짧은 간격으로 순차 추가되어도 된다.
* 텍스트를 드롭하면 텍스트 또는 링크 아이템으로 추가한다.
* 드래그가 보드 영역을 벗어나면 오버레이를 숨긴다.
* 드롭 완료 후 오버레이를 숨긴다.

## 7. 아이템 유형

### 7-1. 공통 카드 구조

* 모든 아이템은 카드 형태로 표시한다.
* 카드 헤더에는 다음을 표시한다.
  * 아바타
  * 작성자명
  * `관리자` 또는 `나` 배지
  * 아이템 타입 아이콘과 타입 라벨
  * 상대 시간
  * 카드 액션
* 카드 액션은 다음을 포함한다.
  * 핀 고정/해제: 관리자에게만 표시
  * 복사
  * 삭제: 관리자 또는 작성자에게 표시
* 카드 본문에는 아이템 타입별 콘텐츠를 표시한다.
* 카드 하단에는 리액션 영역을 표시한다.
* 새로 추가된 카드는 settle 애니메이션으로 등장한다.
* 사용자가 reduced motion을 선호하면 등장 애니메이션을 재생하지 않는다.

### 7-2. 텍스트 아이템

* 텍스트 아이템은 Markdown 서식을 지원한다.
* 지원 Markdown:
  * H1-H4 헤딩
  * 굵게
  * 기울임
  * 취소선
  * 인라인 코드
  * 코드 블록
  * 코드 블록 언어 레이블
  * 링크
  * 순서 없는 목록
  * 순서 있는 목록
  * 체크리스트
  * 인용문
  * 표
  * 수평선
* 코드 블록은 어두운 배경과 상단 언어 바를 가진다.
* 체크리스트의 완료 항목은 체크 표시와 취소선 스타일을 가진다.
* 원시 HTML은 이스케이프하거나 제거해 실행되지 않도록 한다.

### 7-3. 링크 아이템

* URL만 붙여넣으면 링크 아이템으로 생성된다.
* 일반 링크 카드는 다음을 표시한다.
  * 대표 이미지 영역 또는 플레이스홀더
  * 사이트명
  * 제목
  * 설명
  * 원본 URL
* 링크 카드 전체를 클릭하면 새 탭으로 열린다.
* 외부 링크는 안전한 새 탭 열기 속성을 적용한다.
* 대표 이미지를 실제로 불러오지 못하면 패턴 플레이스홀더를 표시할 수 있다.
* 메타데이터 수집 실패 시 원본 URL은 반드시 남긴다.

### 7-4. YouTube 링크 아이템

* YouTube URL은 일반 링크가 아니라 영상형 링크 카드로 표시한다.
* 지원 URL 형태:
  * `youtube.com/watch?v=...`
  * `youtu.be/...`
  * `youtube.com/embed/...`
* 영상형 링크는 다음을 표시한다.
  * 썸네일 또는 썸네일 플레이스홀더
  * 중앙 재생 버튼 오버레이
  * `youtube.com` 배지
  * 영상 제목
* 클릭 시 새 탭에서 원본 YouTube URL을 연다.

### 7-5. 파일 아이템

* 파일은 드래그 앤 드롭 또는 전역 붙여넣기로 추가한다.
* 이미지 파일은 이미지형 카드로 표시한다.
* 이미지형 카드는 다음을 표시한다.
  * 이미지 미리보기 또는 플레이스홀더
  * 이미지 아이콘
  * 파일명
  * 파일 크기
  * 다운로드 버튼
* 비이미지 파일은 문서형 카드로 표시한다.
* 문서형 카드는 다음을 표시한다.
  * 확장자 배지
  * 파일명
  * MIME 타입
  * 파일 크기
  * 다운로드 버튼
* 파일 크기는 B, KB, MB 단위로 읽기 쉽게 표시한다.
* 업로드 크기 제한은 기본 20MB다.

### 7-6. 투표 아이템

* 투표는 별도 아이템 타입으로 입력하지 않고 텍스트 아이템 안의 poll fence로 작성한다.
* 문법:

````md
```poll
1. 라이브 코딩 세션
2. 코드 리뷰 중심
3. 개인 과제 + Q&A
```
````

* poll fence는 ` ```poll `로 시작하고 ` ``` `로 끝난다.
* 항목은 번호 목록 또는 글머리 기호 목록 형태로 작성한다.
* 항목은 최대 10개까지 렌더링한다.
* 각 항목에는 숫자 이모지를 배정한다.
  * 1️⃣, 2️⃣, 3️⃣, 4️⃣, 5️⃣, 6️⃣, 7️⃣, 8️⃣, 9️⃣, 🔟
* 투표 블록 헤더에는 투표 아이콘과 `투표` 라벨을 표시한다.
* 사용자는 한 번에 하나의 항목만 선택할 수 있다.
* 다른 항목을 클릭하면 기존 선택이 자동 해제되고 새 항목이 선택된다.
* 선택된 항목을 다시 클릭하면 선택이 취소된다.
* 각 항목에는 득표 수와 퍼센트를 표시한다.
* 득표 비율은 배경 progress bar로 시각화한다.
* 선택한 항목은 강조 스타일로 표시한다.
* 하단에는 총 득표 수를 표시한다.
* 아직 투표가 없으면 `아직 투표가 없어요`를 표시한다.
* 사용자가 선택한 상태에서는 `선택됨 · 다시 누르면 취소` 힌트를 표시한다.
* 투표 이모지는 리액션 데이터로 활용하되 일반 리액션 pill 목록에서는 필터링한다.

## 8. 피드 표시

### 8-1. 그룹화

* 아이템은 날짜별 세션으로 그룹화한다.
* 세션 헤더에는 날짜 레이블과 아이템 수를 표시한다.
* 날짜 레이블 규칙:
  * 오늘: `오늘`
  * 어제: `어제`
  * 올해 날짜: `n월 n일 (요일)`
  * 올해가 아닌 날짜: `yyyy년 n월 n일 (요일)`
* mockup 화면에서는 차시 레이블을 함께 사용할 수 있다.
  * 예: `1차시`
* 세션 헤더에는 날짜/차시 pill, 구분선, 아이템 수를 표시한다.

### 8-2. 정렬

* 날짜 그룹은 최신 날짜가 위에 온다.
* 같은 그룹 안에서는 최신 아이템이 먼저 표시된다.
* 새 아이템 추가 후 피드는 하단으로 자동 스크롤한다.

### 8-3. 레이아웃

* 기본 피드는 중앙 정렬된 단일 컬럼으로 표시한다.
* Tweaks에서 2열을 선택하면 넓은 화면에서 2열 카드 레이아웃을 사용할 수 있다.
* 화면 폭이 1200px 이하이면 2열 설정이어도 1열로 표시한다.
* 일반 피드 모드에서는 카드 사이 간격을 둔다.
* stack 모드에서는 카드가 살짝 겹쳐 쌓이고 hover 시 펼쳐지는 느낌을 준다.

### 8-4. 끝 표시

* 피드 최하단에는 `— 더미의 끝 —` 텍스트를 표시한다.
* 끝 표시 텍스트는 작고 흐린 스타일로 표시한다.

## 9. 고정(Pin)

### 9-1. 권한과 동작

* 관리자는 아이템을 상단에 고정할 수 있다.
* 관리자는 고정된 아이템을 다시 눌러 고정을 해제할 수 있다.
* 고정 버튼은 관리자에게만 표시한다.
* 고정 성공 시 토스트를 표시한다.
  * 문구: `상단에 고정했어요`
* 고정 해제 시 토스트를 표시한다.
  * 문구: `고정을 해제했어요`
* 아이템 삭제 시 해당 아이템은 고정 목록에서도 제거된다.

### 9-2. 고정 섹션

* 고정된 아이템이 없으면 고정 섹션을 표시하지 않는다.
* 고정된 아이템이 있으면 피드 일반 세션보다 위에 고정 섹션을 표시한다.
* 섹션 헤더에는 핀 아이콘과 `고정된 게시물` 라벨을 표시한다.
* 여러 고정 아이템이 있으면 카드 더미 형태로 겹쳐 보인다.
* 뒤쪽 카드는 살짝 회전되어 노출된다.
* 현재 카드가 앞에 표시된다.
* 여러 고정 아이템이 있으면 이전/다음 버튼을 제공한다.
  * 이전: `‹`
  * 다음: `›`
* 현재 위치를 `n / 전체` 형식으로 표시한다.

## 10. 리액션

### 10-1. 리액션 영역

* 모든 아이템 카드 하단에 리액션 영역을 제공한다.
* 리액션이 하나도 없어도 `+` 버튼은 표시한다.
* 기존 리액션 pill에는 이모지와 카운트를 표시한다.
* 내가 선택한 리액션은 강조 스타일로 표시한다.
* 기존 리액션 pill을 클릭하면 내 리액션을 토글한다.
  * 선택하지 않은 상태면 추가
  * 이미 선택한 상태면 제거

### 10-2. 이모지 피커

* `+` 버튼을 클릭하면 이모지 피커를 표시한다.
* 피커는 12개 프리셋 이모지를 제공한다.
  * 👍
  * ❤️
  * 🔥
  * 😂
  * 👀
  * ✅
  * 💡
  * 🤔
  * 🎉
  * 😮
  * 🙏
  * ⭐
* 피커 외부를 클릭하면 피커를 닫는다.
* 피커에서 이모지를 선택하면 리액션을 추가하고 피커를 닫는다.
* 투표 항목에 쓰이는 숫자 이모지는 일반 리액션 목록에서 숨긴다.

## 11. 실시간 멀티플레이어

### 11-1. 연결

* 실시간 연결은 Trystero 기반 P2P WebRTC 방식을 요구사항으로 둔다.
* 연결 app id는 `pile-board-v1`이다.
* 같은 보드에 접속한 사용자는 같은 room에 연결된다.
* room id는 board id를 따른다.
* 연결 상태는 상단바 라이브 뱃지로 표시한다.
* 연결 중 상태에서는 `연결 중`을 표시한다.
* 연결 중 상태에서는 live dot이 점멸하지 않는다.
* 연결됨 상태에서는 현재 접속자 수를 `n명`으로 표시한다.
* 연결됨 상태에서는 초록 계열 live dot을 점멸 표시한다.

### 11-2. 접속자 표시

* 현재 접속 중인 다른 사용자를 상단바 peer pip으로 표시한다.
* peer pip은 최대 6명까지 개별 표시한다.
* 6명을 초과하면 `+n`으로 초과 인원을 표시한다.
* peer pip의 title 또는 접근 가능한 이름으로 사용자명을 제공한다.

### 11-3. 실시간 커서

* 다른 사용자의 마우스 커서를 화면에 표시한다.
* 커서 위치는 문서 기준 Y 좌표를 사용해 피드 스크롤 오프셋을 반영한다.
* 커서 이동 브로드캐스트는 rAF 기준으로 스로틀한다.
* 화면 밖 커서는 렌더링하지 않는다.
* 커서에는 사용자별 색상을 적용한다.
* 커서 옆에는 사용자명 레이블을 표시한다.
* 커서 레이어는 사용자 입력을 가로막지 않아야 한다.

### 11-4. 실시간 아이템 공유

* 사용자가 새 아이템을 추가하면 같은 보드의 다른 접속자에게 브로드캐스트한다.
* 수신한 아이템은 로컬에서 생성한 아이템과 동일하게 피드에 추가한다.
* 수신한 새 아이템도 필요한 경우 등장 애니메이션과 자동 스크롤 규칙을 따른다.

### 11-5. 이름 공유

* 사용자가 표시 이름을 변경하면 같은 보드의 다른 접속자에게 즉시 전파한다.
* 새 peer가 입장하면 기존 사용자는 자신의 현재 이름을 새 peer에게 전송한다.

## 12. 외관과 테마(Tweaks)

### 12-1. Tweaks 패널

* 우하단에 Tweaks 패널을 제공한다.
* 패널은 열고 닫을 수 있다.
* 패널은 화면 안에서 이동 가능해야 한다.
* 패널 내용이 길면 내부 스크롤을 제공한다.
* 설정 변경은 즉시 화면에 반영된다.
* 설정은 localStorage에 영구 저장되어 새로고침 후에도 유지된다.
* 기본값은 다음과 같다.
  * 강조색: clay
  * 질감: on
  * 쌓는 방식: stack
  * 열 수: two
  * 밀도: comfortable

### 12-2. 강조색

* 강조색은 4개 옵션을 제공한다.
  * clay: 기본값
  * ink
  * sage
  * blue
* 강조색은 활성 채널, 버튼, 체크박스, 고정 상태, 링크 강조, 투표 선택 등에 반영된다.

### 12-3. 질감

* texture on/off 설정을 제공한다.
* on일 때 배경에 도트와 사선 패턴을 적용한다.
* off일 때는 평면 배경을 사용한다.

### 12-4. 쌓는 방식

* `feed`: 카드 사이 간격이 있는 일반 피드.
* `stack`: 카드가 살짝 겹쳐 쌓이는 더미 느낌.
* stack 모드에서 카드 hover 시 카드가 앞으로 들리고 간격이 살짝 열린다.

### 12-5. 열 수

* `one`: 1열 피드.
* `two`: 넓은 화면에서 2열 피드.
* 1200px 이하 화면에서는 항상 1열을 사용한다.

### 12-6. 밀도

* `comfortable`: 기본 패딩과 글자 크기.
* `compact`: 카드 패딩과 글자 크기를 줄인 조밀한 모드.

## 13. 알림과 피드백

### 13-1. 토스트

* 토스트는 화면 하단 중앙에 표시한다.
* 토스트는 약 2.2초 후 자동으로 사라진다.
* 토스트에는 아이콘과 메시지를 함께 표시할 수 있다.
* 토스트는 다음 작업 후 표시한다.
  * 텍스트 올림
  * 링크 올림
  * 파일 올림
  * 이미지 붙여넣기
  * 고정
  * 고정 해제
  * 삭제
  * 복사
  * 보드 URL 복사
  * 관리자 모드 켬
  * 관리자 모드 끔
  * 채널 생성

### 13-2. 주요 토스트 문구

* `텍스트를 올렸어요`
* `링크를 올렸어요`
* `<n>개 파일을 올렸어요`
* `이미지를 붙여넣었어요`
* `상단에 고정했어요`
* `고정을 해제했어요`
* `삭제했어요`
* `파일과 함께 삭제했어요`
* `클립보드에 복사했어요`
* `보드 URL을 복사했어요`
* `관리자 모드 · 모든 아이템 관리 가능`
* `관리자 모드를 껐어요`
* `#<채널명> 채널을 만들었어요`

### 13-3. 복사

* 아이템 복사 버튼을 누르면 아이템 타입에 따라 복사 값을 결정한다.
  * 텍스트: 본문
  * 링크: 원본 URL
  * 파일: 파일명
* 복사 성공 여부와 무관하게 사용자가 명확한 피드백을 받을 수 있도록 토스트를 표시한다.

## 14. 빈 상태

* 채널에 아이템이 없으면 빈 상태 UI를 표시한다.
* 빈 상태는 더미 일러스트, 제목, 안내 문구로 구성한다.
* 제목: `아직 더미가 비어 있어요`
* 안내 문구는 붙여넣기 또는 파일 드롭으로 첫 자료를 쌓을 수 있음을 설명한다.
* 빈 상태에서도 컴포저는 표시되어야 한다.
* 빈 상태 아래에도 피드 끝 표시를 유지한다.

## 15. 화면 스타일 요구사항

### 15-1. 전체 톤

* 전체 UI는 밝고 따뜻한 종이색 계열 배경을 사용한다.
* 카드와 버튼은 낮은 대비의 테두리와 부드러운 그림자를 사용한다.
* UI는 과도한 장식보다 자료를 읽기 쉽게 하는 데 집중한다.
* “pile” 은유는 로고, 빈 상태, stack 모드, 고정 카드 더미에서 드러낸다.

### 15-2. 상단바

* 상단바는 고정 높이의 가로 레이아웃이다.
* 왼쪽에는 로고와 보드 URL이 있다.
* 오른쪽에는 peer pips, live badge, 사용자 chip, 관리자 버튼이 있다.
* 상단바는 하단 테두리로 피드와 구분한다.

### 15-3. 카드

* 카드는 둥근 모서리와 약한 그림자를 가진다.
* hover 시 살짝 들어 올려지는 느낌을 준다.
* stack 모드에서는 카드가 약간 회전하거나 겹쳐 보일 수 있다.
* 카드 내부 텍스트는 작은 화면에서도 넘치지 않아야 한다.

### 15-4. 플레이스홀더

* 링크 대표 이미지, YouTube 썸네일, 이미지 미리보기는 실제 리소스가 없을 때 패턴 플레이스홀더로 대체할 수 있다.
* 플레이스홀더에는 용도를 설명하는 짧은 라벨을 표시한다.
  * 예: `대표 이미지 · og:image`
  * 예: `YouTube 임베드 · video embed`

## 16. 접근성

* 장식용 아이콘은 보조기술에서 불필요하게 읽히지 않도록 처리한다.
* 의미 있는 버튼에는 title, aria-label 또는 명확한 텍스트 라벨을 제공한다.
* Tweaks 패널의 토글은 switch 역할과 checked 상태를 제공한다.
* Tweaks의 segmented control은 radiogroup/radio 역할을 제공한다.
* 새 탭 링크는 안전한 속성을 적용한다.
* 키보드 상호작용을 지원한다.
  * 채널 추가: Enter 확정, Escape 취소
  * 이름 편집: Enter 확정, Escape 취소
  * 컴포저: Cmd/Ctrl+Enter 제출, Escape 취소
* 애니메이션은 `prefers-reduced-motion`을 존중한다.
* 커서 레이어와 오버레이는 필요 이상으로 입력을 방해하지 않아야 한다.

---

## 17. 성능과 반응성

* 피드 스크롤은 부드럽게 동작해야 한다.
* 새 아이템 추가 시 자동 스크롤은 사용자가 맥락을 잃지 않도록 자연스럽게 수행한다.
* 실시간 커서 전송은 rAF로 스로틀한다.
* 화면 밖 원격 커서는 렌더링하지 않는다.
* 채널 탭은 많은 채널이 있어도 가로 스크롤로 사용할 수 있어야 한다.
* 2열 피드는 넓은 화면에서만 적용하고 좁은 화면에서는 1열로 단순화한다.

---

## 18. 안전 요구사항

* 원시 HTML은 Markdown 렌더링 결과에 실행 가능한 형태로 남지 않아야 한다.
* 외부 링크는 새 탭으로 열고 opener 공격을 방지해야 한다.
* 파일 접근은 허용된 파일에만 가능해야 한다.
* 파일 크기 제한을 초과한 업로드는 거부해야 한다.
* 사용자가 삭제 권한이 없는 아이템은 삭제할 수 없어야 한다.
* 관리자가 아닌 사용자는 핀 고정/해제 기능을 사용할 수 없어야 한다.
* 관리자 상태는 클라이언트 표시만으로 신뢰하지 않고 서버에서 검증되어야 한다.

## 19. 검증 시나리오

### 19-1. 보드와 채널

* 새 보드 URL에 접속하면 보드 화면이 열린다.
* 첫 텍스트 아이템을 올리면 보드와 기본 채널 4개가 준비된다.
* 상단바 URL 버튼을 클릭하면 URL 복사 토스트가 표시된다.
* 기본 채널 탭이 일반 / 과제 / Q&A / 자료실 순서로 표시된다.
* 각 채널 탭에 해당 채널 아이템 수가 표시된다.
* 채널을 전환하면 해당 채널 아이템만 표시된다.
* 관리자로 새 채널을 추가하면 새 채널로 자동 전환된다.

### 19-2. 작성과 붙여넣기

* 접힌 컴포저를 클릭하면 텍스트에리어가 열린다.
* 빈 내용일 때 제출 버튼이 비활성화된다.
* 텍스트를 입력하고 Cmd/Ctrl+Enter를 누르면 텍스트 아이템이 생성된다.
* Escape를 누르면 컴포저가 닫히고 내용이 버려진다.
* URL만 입력하면 링크 아이템이 생성된다.
* input/textarea 밖에서 텍스트를 붙여넣으면 아이템이 즉시 생성된다.
* input/textarea 안에서는 전역 붙여넣기가 동작하지 않는다.
* 클립보드 이미지를 붙여넣으면 파일 아이템이 생성된다.

### 19-3. 파일과 드롭

* 파일을 보드 위로 드래그하면 드롭 오버레이가 표시된다.
* 보드 밖으로 드래그하면 오버레이가 사라진다.
* 파일을 드롭하면 파일 아이템이 생성된다.
* 이미지 파일은 미리보기형 카드로 표시된다.
* PDF 등 비이미지 파일은 확장자 배지형 카드로 표시된다.
* 텍스트를 드롭하면 텍스트 또는 링크 아이템이 생성된다.

### 19-4. Markdown과 투표

* H1-H4, 굵게, 기울임, 취소선, 코드, 링크, 목록, 체크리스트, 인용문, 표, 수평선이 렌더링된다.
* 원시 HTML은 실행되지 않는다.
* poll fence가 있는 텍스트는 투표 블록을 표시한다.
* 투표 항목은 최대 10개까지만 표시된다.
* 항목 클릭 시 득표 수, 퍼센트, progress bar가 갱신된다.
* 다른 항목 클릭 시 기존 선택이 해제된다.
* 선택 항목 재클릭 시 투표가 취소된다.

### 19-5. 카드 액션

* 복사 버튼을 누르면 타입별 값이 복사되고 토스트가 표시된다.
* 작성자는 본인 아이템을 삭제할 수 있다.
* 작성자가 아닌 일반 사용자는 타인 아이템 삭제 버튼을 볼 수 없거나 사용할 수 없다.
* 관리자는 모든 아이템을 삭제할 수 있다.
* 관리자는 아이템을 고정/해제할 수 있다.
* 아이템 삭제 시 고정 목록에서도 제거된다.

### 19-6. 리액션

* `+` 버튼을 누르면 12개 이모지 피커가 열린다.
* 피커 외부 클릭 시 닫힌다.
* 이모지를 선택하면 리액션 pill이 추가된다.
* 같은 이모지를 다시 누르면 내 리액션이 제거된다.
* 내가 누른 리액션은 강조 표시된다.
* 투표 숫자 이모지는 일반 리액션 목록에 표시되지 않는다.

### 19-7. 실시간

* 같은 보드에 두 사용자가 접속하면 연결 상태가 표시된다.
* 접속자 수가 `n명`으로 표시된다.
* 다른 사용자의 peer pip이 상단바에 표시된다.
* 다른 사용자의 커서가 이름표와 함께 표시된다.
* 다른 사용자가 새 아이템을 추가하면 내 피드에도 추가된다.
* 다른 사용자가 이름을 바꾸면 표시 이름이 갱신된다.

### 19-8. Tweaks와 반응형

* 강조색 변경이 즉시 반영된다.
* texture on/off가 배경에 반영된다.
* feed/stack 모드가 카드 배치에 반영된다.
* one/two 열 설정이 반영된다.
* 1200px 이하에서는 two 설정이어도 1열로 표시된다.
* comfortable/compact 밀도 변경이 카드 크기에 반영된다.
* 설정은 새로고침 후에도 유지된다.

### 19-9. 빈 상태와 끝 표시

* 아이템이 없는 채널에는 빈 상태 UI가 표시된다.
* 빈 상태에는 더미 일러스트, 제목, 안내 문구가 있다.
* 피드 최하단에는 `— 더미의 끝 —`이 표시된다.

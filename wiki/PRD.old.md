# pile — Product Requirements (v0.2-current)

> 로그인 없이 과제물·링크·교재를 던져놓는 온라인 보드. 빨래 더미(pile) 은유 기반.

이 문서는 현재 구현을 기준으로 pile의 제품 요구사항과 동작 계약을 기록한다. 향후 기능은 로드맵에만 분리해 적고, 본문은 지금 사용자가 경험하는 방식을 정본으로 둔다.

---

## 1) 목표 및 범위

* **목표**: 오프라인/온라인 모임 중 참가자들이 로그인 없이 자료를 빠르게 공유·수집할 수 있는 보드 제공.
* **제품 범위**: URL 기반 보드, 채널, 익명 사용자, 관리자 로그인, 채널 생성, 아이템 생성/삭제, Markdown 렌더링, 링크 프리뷰, 파일 업로드.
* **사용 흐름**: 참가자는 공유받은 `/:boardId` URL에 들어와 텍스트/링크/파일을 붙여넣거나 드롭한다. 보드가 아직 없으면 첫 아이템 생성 시 자동으로 보드와 기본 채널이 만들어진다.
* **비범위**: 계정 시스템, 권한 세분화, 외부 스토리지 연동, 모바일 앱, WYSIWYG 편집, 다중 서버 실시간 동기화.

---

## 2) 사용자와 권한

### 익명 사용자

* 별도 로그인 없이 보드에 입장한다.
* 처음 접근하면 익명 식별자와 닉네임이 발급된다.
* 표시 이름을 직접 설정하거나 비울 수 있다.
* 텍스트, 링크, 파일 아이템을 생성할 수 있다.
* 본인이 올린 아이템을 삭제할 수 있다.

### 관리자

* 관리자 화면에서 비밀번호로 관리자 모드를 활성화한다.
* 새 채널을 만들 수 있다.
* 모든 아이템을 삭제할 수 있다.
* 관리자 표시명은 설정된 이름으로 노출된다.

---

## 3) 도메인 모델

* **Board**: 모임 단위 워크스페이스. URL에서는 `/:boardId` 형식으로 접근한다.
* **Channel**: 보드 하위 분류. URL에서는 `/:boardId/:channelId` 형식으로 접근한다.
* **Item**: 채널 안에 올라오는 자료. `text`, `file`, `link` 중 하나다.
* **AnonUser**: 로그인 없이 사용자를 구분하기 위한 익명 사용자. 시스템 닉네임과 선택 표시 이름을 가진다.
* **BoardMember**: 보드 참여 기록을 위한 개념. 현재 핵심 화면에서는 직접 노출하지 않는다.

---

## 4) 데이터 규칙

### 세션(차시)

* 보드는 세션 기준 시각과 세션 길이를 가진다.
* 아이템 생성 시 현재 시각이 속한 세션 시작 시각을 저장한다.
* 보드 화면은 아이템을 세션별로 묶어 보여준다.
* 세션 기준값이 유효하지 않으면 세션 미지정 상태로 처리한다.

### 보드와 기본 채널

* 존재하지 않는 보드 URL에서도 화면에 진입할 수 있다.
* 해당 화면에서 첫 아이템을 붙여넣으면 보드가 자동 생성된다.
* 자동 생성 보드의 이름은 URL의 board id를 따른다.
* 기본 채널은 `일반`이며, 주소용 값은 `default`다.

### 익명 사용자

* 최초 접근 또는 첫 아이템 생성 시 익명 사용자가 준비된다.
* 닉네임은 서버가 생성한다.
* 표시 이름은 사용자가 수정할 수 있다.
* 표시 이름이 비어 있으면 시스템 닉네임을 보여준다.

---

## 5) 화면과 라우트

* `/` — 제품 소개 페이지.
* `/admin` — 관리자 로그인/로그아웃 화면.
* `/:boardId` — 보드 기본 채널 화면. 보드가 없으면 첫 붙여넣기로 생성 가능한 화면을 보여준다.
* `/:boardId/:channelId` — 특정 채널 화면. 보드나 채널이 없으면 찾을 수 없음 화면으로 처리한다.

### 보드 화면

* 상단에서 보드 URL을 복사할 수 있다.
* 현재 사용자 이름 또는 관리자 상태를 확인할 수 있다.
* 채널 탭으로 채널을 이동할 수 있다.
* 관리자는 새 채널을 추가할 수 있다.
* 사용자는 붙여넣기, 수동 붙여넣기 버튼, 드래그 앤 드롭으로 자료를 올릴 수 있다.
* 아이템 목록은 세션별로 묶인다.
* 각 아이템에는 작성자, 작성 시점, 삭제/복사 액션이 표시된다.

---

## 6) 아이템 유형

### 텍스트

* 일반 텍스트와 Markdown을 그대로 저장한다.
* 화면에서는 Markdown을 렌더링한다.
* 코드 블록, 표, 체크리스트 등 기본 Markdown 확장 문법을 지원한다.

### 링크

* 붙여넣은 URL을 링크 아이템으로 저장한다.
* 가능한 경우 제목, 설명, 대표 이미지를 수집해 프리뷰로 보여준다.
* YouTube 링크는 동영상 임베드로 표시한다.
* 메타 정보를 가져오지 못해도 원본 URL은 남긴다.

### 파일

* 파일 드롭 또는 클립보드 이미지 붙여넣기를 지원한다.
* 파일명, 형식, 크기를 함께 보여준다.
* 이미지는 미리보기로 보여준다.
* 그 외 파일은 다운로드 링크로 제공한다.
* 업로드 크기 제한은 기본 20MB다.

---

## 7) 삭제와 갱신

* 작성자는 본인이 올린 아이템을 삭제할 수 있다.
* 관리자는 모든 아이템을 삭제할 수 있다.
* 파일 아이템을 삭제하면 연결된 파일도 함께 삭제를 시도한다.
* 붙여넣기, 삭제, 채널 생성 후 화면은 최신 상태로 다시 갱신된다.

---

## 8) 안전장치

* 관리자 모드는 서명된 관리자 상태를 확인한 뒤 활성화된다.
* Markdown 렌더링 시 위험한 HTML은 제거한다.
* 업로드 파일은 허용된 저장 영역 안에서만 접근할 수 있다.
* 링크는 웹 URL만 허용한다.
* 파일 업로드에는 크기 제한을 적용한다.
* MIME/확장자 허용 목록과 레이트 리밋은 로드맵 항목이다.

---

## 9) 검증 시나리오

* `/admin`에서 로그인/로그아웃한다.
* 새 보드 URL에 접속해 텍스트를 붙여넣고 보드와 기본 채널이 자동 생성되는지 확인한다.
* 같은 보드에서 링크를 붙여넣고 프리뷰 또는 URL fallback을 확인한다.
* 이미지/파일을 드롭하고 미리보기 또는 다운로드 링크를 확인한다.
* 작성자 계정으로 본인 아이템을 삭제한다.
* 관리자 계정으로 다른 사용자의 아이템을 삭제한다.
* 관리자 계정으로 새 채널을 생성하고 해당 채널 화면에 접근한다.
* 위험한 Markdown/HTML 입력이 안전하게 렌더링되는지 확인한다.
* 허용되지 않은 파일 접근 시도가 차단되는지 확인한다.

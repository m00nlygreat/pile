# pile — 요구사항 정의서

> 소규모 그룹(스터디, 팀, 모임)이 함께 자료를 빠르게 쌓고 공유하는 실시간 협업 보드 앱.

---

## 1. 보드 · 채널

- 보드는 고유 슬러그(`pile.so/<board-id>`)로 식별된다
  - 상단바에 보드 URL이 표시되며, 클릭 시 클립보드에 복사된다
  - 복사 성공 시 토스트 알림이 표시된다
- 보드 안에 여러 **채널**이 존재한다
  - 기본 채널: 일반 / 과제 / Q&A / 자료실
  - 채널은 탭(chip) 형태로 상단에 나열된다
  - 각 채널 탭에 아이템 수(카운트 배지)가 표시된다
  - 채널 전환 시 해당 채널의 피드만 표시된다
- 관리자는 채널을 새로 추가할 수 있다
  - 채널 추가 버튼 클릭 → 인라인 입력 → Enter 확정 / Escape 취소
  - 생성 직후 해당 채널로 자동 전환된다

---

## 2. 아이템 타입

### 2-1. 텍스트 (text)
- Markdown 서식을 지원한다
  - 헤딩(H1–H4), 굵게(`**`), 기울임(`*`), 취소선(`~~`)
  - 인라인 코드(`` ` ``), 코드 블록(` ``` ` — 언어 레이블 포함)
  - 링크(`[텍스트](url)`)
  - 순서 없는 목록 / 순서 있는 목록
  - 체크리스트(`- [x]` / `- [ ]`)
  - 인용문(`>`)
  - 표(pipe table)
  - 수평선(`---`)
- 원시 HTML은 이스케이프되어 XSS를 방지한다

### 2-2. 링크 (link)
- 텍스트 입력란에 URL만 붙여넣으면 자동으로 링크 아이템으로 분류된다
- 일반 링크: OG 대표 이미지 플레이스홀더 + 사이트명 + 제목 + 설명 + URL 표시
- YouTube 링크: 썸네일 플레이스홀더 + 재생 버튼 오버레이 + 영상 제목 표시
- 링크 카드 클릭 시 새 탭으로 열린다

### 2-3. 파일 (file)
- 드래그 앤 드롭 또는 전역 붙여넣기로 파일을 올릴 수 있다
- 이미지 파일: 미리보기 플레이스홀더 + 파일명 + 용량 + 다운로드 버튼
- 비이미지 파일(PDF 등): 확장자 뱃지 + 파일명 + MIME 타입 + 용량 + 다운로드 버튼

### 2-4. 투표 (poll)
- 텍스트 아이템 안에 ` ```poll ... ``` ` 펜스를 작성하면 투표 블록으로 렌더링된다
  - 항목 최대 10개
  - 각 항목에 숫자 이모지(1️⃣–🔟) 배정
- 투표 동작
  - 한 번에 하나의 항목만 선택 가능 (다른 항목 클릭 시 기존 선택 자동 해제)
  - 선택된 항목 재클릭 시 취소
  - 각 항목의 득표 수 및 퍼센트(%) 실시간 표시
  - 배경 바(progress bar)로 비율 시각화
  - 총 득표 수 및 "선택됨 · 다시 누르면 취소" 힌트 표시

---

## 3. 컴포저 (글 올리기)

- 피드 상단에 항상 고정(sticky)으로 표시된다
- **접힌 상태**: 붙여넣기 힌트 버튼으로 표시
- **펼친 상태**: 텍스트에리어 + 제출 버튼
  - `⌘↵` (Cmd+Enter) 단축키로 제출
  - `Escape` 로 취소
  - 내용 없을 때 제출 버튼 비활성화
- 제출 시 내용에 따라 자동 분류
  - URL → 링크 아이템
  - 그 외 텍스트 → 텍스트 아이템
- 전역 붙여넣기(페이지 어디서나 `⌘V`)
  - 텍스트에리어·인풋이 포커스된 상태가 아닌 경우에만 동작
  - 클립보드 이미지 → 파일 아이템으로 즉시 추가
  - 클립보드 텍스트/URL → 텍스트 또는 링크 아이템으로 즉시 추가

---

## 4. 드래그 앤 드롭

- 파일을 보드 위로 드래그하면 드롭 오버레이가 표시된다
- 드롭 시 파일 아이템으로 추가된다
- 텍스트 드래그 앤 드롭도 지원한다
- 오버레이는 마우스가 보드를 벗어나면 자동으로 사라진다

---

## 5. 피드 표시

- 아이템은 **날짜별 세션**으로 그룹화된다
  - 오늘 / 어제 / n월 n일 (요일) / 연도 포함(올해가 아닐 경우)
  - 각 세션 헤더에 날짜 레이블과 아이템 수 표시
- 각 아이템은 **카드** 형태로 표시된다
  - 카드 헤더: 아바타 + 작성자명 + 관리자/나 뱃지 + 아이템 타입 아이콘 + 상대 시간
  - 카드 액션(호버 시): 핀 고정, 복사, 삭제
  - 카드 하단: 리액션 영역
- 새 아이템 추가 시 피드 하단으로 자동 스크롤
- 새로 추가된 아이템은 settle 애니메이션으로 등장 (`prefers-reduced-motion` 존중)

---

## 6. 고정(Pin)

- 관리자가 카드를 상단에 고정할 수 있다
- 고정된 아이템은 피드 최상단 **고정 섹션**에 별도 표시된다
  - 카드 더미(pile) 형태로 겹쳐 보임 (뒤 카드가 살짝 기울어져 노출)
  - 여러 고정 아이템이 있을 때 ‹ / › 버튼으로 탐색 가능
  - `n / 전체` 인디케이터 표시
- 카드 핀 버튼을 다시 누르면 고정이 해제된다
- 아이템 삭제 시 고정 목록에서도 자동 제거된다
- 토스트 알림: "상단에 고정했어요" / "고정을 해제했어요"

---

## 7. 리액션

- 모든 아이템 카드 하단에 리액션 영역이 있다
- 기존 리액션 pill 클릭 → 토글(추가/제거)
- `+` 버튼 클릭 → 이모지 피커 팝업(12개 프리셋) 표시
- 피커 외부 클릭 시 자동 닫힘
- 내가 선택한 리액션은 강조 스타일(mine)로 표시된다
- 투표 아이템에서는 투표 이모지(1️⃣–🔟)가 리액션으로 활용되며 별도 필터링된다

---

## 8. 사용자 · 권한

### 8-1. 일반 사용자
- 익명 닉네임으로 참여 (기본값: 동물 닉네임)
- 상단바 닉네임 클릭 → 인라인 편집 → Enter 확정 / Escape 취소
- 자신이 올린 아이템만 삭제 가능

### 8-2. 관리자 모드
- 상단바 "관리자 로그인" 버튼으로 토글
- 관리자 활성화 시 버튼이 강조 표시됨
- 관리자 권한
  - 모든 아이템 삭제 가능
  - 아이템 고정 / 고정 해제
  - 새 채널 추가
- 관리자 아바타에 방패(shield) 아이콘, 카드에 "관리자" 뱃지 표시

---

## 9. 실시간 멀티플레이어

- Trystero 기반 P2P WebRTC 연결 (`appId: "pile-board-v1"`)
- 같은 보드(roomId) 접속자끼리 자동 연결
- 연결 상태 표시
  - 상단바 라이브 뱃지: "연결 중" (점멸 없음) / "n명" (초록 점멸)
- 현재 접속 중인 다른 사용자 아바타 pip이 상단바에 표시됨 (최대 6개, 초과 시 +n 표시)
- **실시간 커서 공유**
  - 마우스 이동 시 rAF 스로틀로 위치 브로드캐스트
  - 피드 스크롤 오프셋 포함한 문서 기준 Y 좌표 전송
  - 화면 밖 커서는 렌더링 생략
  - 커서 옆에 사용자명 레이블 표시
- **실시간 아이템 공유**
  - 새 아이템 추가 시 같은 방 피어에게 브로드캐스트
  - 수신 아이템은 로컬과 동일하게 피드에 추가됨
- **이름 공유**
  - 닉네임 변경 시 모든 피어에게 즉시 전파
  - 새 피어 입장 시 현재 이름 전송

---

## 10. 외관 · 테마 (Tweaks)

- 우하단 Tweaks 패널을 통해 실시간으로 변경 가능
- **강조색 (accent)**: clay(기본) / ink / sage / blue 4가지 스와치
- **질감 (texture)**: on(도트+사선 배경 패턴) / off
- **쌓는 방식 (pileMode)**
  - `feed`: 카드 사이 간격 있는 일반 피드
  - `stack`: 카드가 살짝 겹쳐 쌓이는 더미 느낌, hover 시 펼쳐짐
- **열 수 (columns)**: 1열 / 2열 (1200px 이하에서는 항상 1열)
- **밀도 (density)**: comfortable(기본) / compact(패딩·폰트 축소)
- 설정은 localStorage에 영구 저장된다

---

## 11. 알림 · 피드백

- **토스트 알림**: 작업 완료 후 하단 중앙에 2.2초간 표시
  - 텍스트 올림 / 링크 올림 / 파일 올림 / 이미지 붙여넣기
  - 고정 / 고정 해제 / 삭제 / 복사 / URL 복사
  - 관리자 모드 켬/끔 / 채널 생성
- **드롭 오버레이**: 파일 드래그 시 전체 화면 오버레이로 드롭 영역 안내

---

## 12. 아바타

- 사용자명 기반 결정론적 색상 생성 (warm 20–80° 색조)
- 일반 사용자: 이름 첫 글자
- 관리자: 방패(shield) 아이콘

---

## 13. 빈 상태 · 끝 표시

- 채널에 아이템이 없으면 빈 상태 UI 표시
  - 더미(pile) 일러스트 + "아직 더미가 비어 있어요" + 안내 문구
- 피드 최하단에 "— 더미의 끝 —" 텍스트 표시

---

## 14. 접근성 · 성능

- 카드 settle 애니메이션은 `prefers-reduced-motion: no-preference` 조건 하에서만 재생
- 아이콘은 `aria-hidden="true"` 처리
- 채널 스크롤바 숨김(scrollbar-width: none)
- rAF 스로틀로 커서 브로드캐스트 성능 최적화
- 컴포저, 피커 등 인풋 외부 클릭 닫힘 처리
